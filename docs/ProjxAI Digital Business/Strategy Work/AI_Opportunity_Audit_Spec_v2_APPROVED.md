## Session 24 Decisions (1 May 2026)

### Technical Architecture — Streaming (CONFIRMED)

**Decision:** Claude API call uses streaming (`messages.stream()`) instead of a blocking `messages.create()` call.

**Why:** Vercel Hobby plan has a hard 10-second function timeout. `export const maxDuration = 60` has no effect on Hobby. Sonnet takes 15–30s. Streaming bypasses this because the function returns a `ReadableStream` immediately — Vercel's timeout clock stops, and the stream stays alive for the full response duration.

**Implementation:**
- `/api/audit/submit` — creates Supabase session + ABR lookup (fast), then streams Claude tokens to client. Sends `__SESSION__:<id>` as first line so client knows the session ID.
- `/api/audit/save` (new endpoint) — receives completed JSON + sessionId from client, validates fields, writes to `audit_reports`, returns `accessToken`.
- `page.tsx handleSubmit()` — reads stream with `getReader()`, buffers full text, brace-scans for JSON, POSTs to `/api/audit/save`, redirects to report.
- Model: `claude-sonnet-4-6`. max_tokens: 4096. Temperature: 0.3.
- No Vercel Pro upgrade required. No quality compromise.

**Rule going forward:** Never revert to a blocking Claude API call on this route. If a timeout issue recurs, the answer is always to check streaming, not to downgrade the model.

### Question System — Redesign Direction (CONFIRMED IN PRINCIPLE, BUILD IN SESSION 25)

**Decision:** Replace the current 14-question linear wizard with a hybrid branching system.

**Structure:**
- 4–5 universal context questions (business type, team size, revenue model, bottleneck, manual hours) — click-through wizard style, fast and low friction
- 2–3 topic silos branch from the bottleneck answer, each 5–6 questions deep on that specific pain area
- Silos map to: Customer Communication & Service | Admin & Operations | Sales & Lead Management | Finance & Reporting | Marketing & Analytics | People & HR
- Strategic free-text fields at high-value moments only (bottleneck description, specific workflow, "what does a good outcome look like?") — 2–3 max, never consecutive
- Goal: more specific answers → more specific report. "Wow factor" comes from the report reading like someone who actually sat with them.

**UX principle:** Hybrid wizard + selective free text. Not fully conversational chat (overkill), not pure form (too generic). No tax-return feeling — every question must feel relevant to what they already answered.

**Confirmed deferred:** Full conversational/chat UI is a later consideration, not Session 25 scope.

### Post-Report Experience — "At Your Pace" Exploration (CONFIRMED, ADDED TO WORKSPACE PHASE 1)

**Decision:** After receiving their report, users should be able to explore at their own pace — not pushed straight to a booking.

**Feature:** Each opportunity card in the report gets interactive actions: "Tell me more", "Add more context", "I'm interested in finding out more". Clicking opens an AI chat (Claude API, report JSON as context) that goes deeper on that specific opportunity. User can ask clarifying questions, get ROI detail, understand the implementation approach — all without talking to a human until they're ready.

**Escalation path:** When ready, "Talk to someone" → Cal.com booking with full conversation summary attached.

**Why this works:** Respects how people actually buy — they research privately, build confidence, then reach out. By the time they book a call, they're already half-sold and the conversation is pre-qualified.

**Where it lives:** Inside the ProjxAI Workspace (`/portal/*`) — confirmed Option A (inside existing site). NOT a separate product. This is a Phase 1 Workspace feature, building directly on the existing report page and Supabase session.

**Reference:** Strategic Foundation v1.1 Part 6 — already decided. Workspace lives at `/portal/*`, uses Supabase Auth (magic link), report persists permanently (not a one-time link).
